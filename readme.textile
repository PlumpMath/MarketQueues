h4. Запуск


Вероятно, проще всего запускать мавеном. 

  @mvn package@
  @mvn gwt:run@

Возможно, ему потребуется какое-то время, чтобы подтянуть зависимости. Idea замечательно импортирует проект из pom-файла и позволяет запускать проект из-под нее.

h4. Заметки по архитектуре


Раньше с GWT я никогда не работал, однако после прочтения задания я сразу подумал именно о нем: поскольку необходимо было кроссбраузерно организовать на клиенте достаточно интересную визуализацию. 
При беглом осмотре я обнаружил, что GWT дает все необходимое для организации архитектуры. Старался держаться привычных для GWT вьюшек-презентеров, прикручивать для такого приложения MV*-фреймворки было бы оверхэдом на мой взгляд.
Клиент-серверное АПИ тоже сделано "в лоб" для экономии времени.

По TDD делал только фрагменты бизнес-логики, которые так делать было быстрее и проще, поэтому, тестов не много.

 Вьюшка для диаграмм вышла достаточно бойлерплейтной - я состредоточился больше на том, что увидит пользователь. Все-таки это вюшка, этакая "вещь в себе", от устройства которой снаружи мало что зависит.
